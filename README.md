## Ответы на контрольные вопросы

1. git add -p позволяет создавать атомарные коммиты, что делает git bisect максимально эффективным.

2. Lock-файл фиксирует точные версии всех зависимостей, гарантируя идентичное окружение на всех машинах.

3. Makefile - исполняемая документация, которая автоматизирует процессы и устраняет человеческие ошибки.

4. Тесты показывают как использовать код и чего ожидать на выходе, а фиксация seed гарантирует детерминированное поведение при случайных процессах.

5. make install полностью восстановит окружение с нуля, создав новый venv и установив все зависимости - результат будет идентичен исходному
