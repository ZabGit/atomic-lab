# Анализ рисков неатомарных коммитов

1. **Сложность отката** — нельзя отменить только одно изменение, только всё сразу.
2. **Проблемы при код-ревью** — сложно понять, что именно менялось в каждом логическом блоке.
3. **Риск «снежного кома» зависимостей** — одно изменение может случайно затронуть другой функционал.
4. **Git bisect становится бесполезным** — невозможно найти коммит, который внёс конкретную ошибку.
5. **Конфликты при слиянии** — вероятность конфликтов растёт с размером коммита.
